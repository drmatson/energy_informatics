üî¢ NumPy#

üì¶ Installation#

If NumPy is not installed, you can add it using:

pip install numpy

or (if using Anaconda):

conda install numpy

‚ùì What is NumPy?#

NumPy (Numerical Python) is the core library for scientific computing in Python.

  • Provides powerful N-dimensional arrays.

  • Supports vectorized operations (fast math on arrays).

  • Contains tools for statistics, linear algebra, random numbers.

  • Forms the base for many libraries: Pandas, scikit-learn, TensorFlow, PyTorch.

import numpy as np

‚ú® Creating NumPy Arrays#

üëâ Just like a plain list of numbers, NumPy arrays make it easy to store and process energy production values (or any dataset).

From list#

# Daily solar energy production in kWh
energy_data = [120, 135, 150, 160]
energy_arr = np.array(energy_data)

print(energy_arr)
# [120 135 150 160]

[120 135 150 160]

üîé Range, linspace#

  • np.arange(start, stop, step) Generates numbers starting at start, up to (but not including) stop, stepping by step. Useful for time intervals, indexes, or sequences with fixed steps.

  • np.linspace(start, stop, num) Generates num evenly spaced values from start to stop (inclusive). Useful for smooth scales, plotting, or dividing a range evenly.

  • üëâ arange() = ‚Äústeps by interval‚Äù

  • üëâ linspace() = ‚Äúsplit into equal parts‚Äù

hours = np.arange(0, 10, 2)
print("Measurement hours:", hours)
# [0 2 4 6 8]

# np.linspace(start, stop, num)
# Simulate 5 evenly spaced efficiency levels between 0% and 100%
efficiency = np.linspace(0, 1, 5)
print("Efficiency levels:", efficiency)
# [0.   0.25 0.5  0.75 1. ]

Measurement hours: [0 2 4 6 8]
Efficiency levels: [0.   0.25 0.5  0.75 1.  ]

üé≤ Random Numbers in NumPy#

NumPy has several functions to generate random numbers useful for simulations in data science and energy analytics ‚ö°.

üëâ With these, you can model efficiency üåû, noise üìâ, and output ‚ö° to make your simulations more realistic.

üåû np.random.rand(n) ‚Üí Uniform distribution [0, 1)#

Generates n random numbers between 0 and 1, all equally likely. ‚úÖ Use: simulate efficiencies, probabilities, or fractions.

# Example: simulate solar panel efficiency (random values between 0 and 1)
efficiency = np.random.rand(3)
print("Solar efficiency:", efficiency)

Solar efficiency: [0.11107385 0.8043761  0.71664469]

üìâ np.random.randn(n) ‚Üí Standard normal distribution (mean=0, std=1)#

Generates n random numbers with a bell-curve distribution centered at 0. ‚úÖ Use: simulate sensor noise, measurement errors, or fluctuations.

# Example: simulate noise in sensor readings
sensor_noise = np.random.randn(3)
print("Sensor noise:", sensor_noise)

Sensor noise: [ 0.54972407 -0.11752996  0.22364662]

‚ö° np.random.randint(low, high, size) ‚Üí Random integers#

Generates random integers in the range [low, high). ‚úÖ Use: simulate discrete counts, daily power output, or events.

# Example: simulate random daily power output (kWh) between 1 and 10
daily_output = np.random.randint(1, 10, 5)
print("Daily output:", daily_output)

Daily output: [6 2 5 9 2]

üé≤ Why Use seed()?#

  • Ensures consistent results for experiments.

  • Makes it easier to compare models or share notebooks.

  • Useful when teaching or debugging so outputs don‚Äôt change randomly.

üëâ Think of seed() as setting the starting point for randomness üé≤.

# Set seed to make results reproducible
np.random.seed(123)

# Simulate daily solar energy production (MWh) for 7 days
solar_output = np.random.randint(100, 200, 7)

# Simulate daily wind energy production (MWh) for 7 days
wind_output = np.random.randint(150, 250, 7)

print("Solar output (MWh):", solar_output)
print("Wind output (MWh):", wind_output)

Solar output (MWh): [166 192 198 117 183 157 186]
Wind output (MWh): [247 246 197 223 182 196 246]

üîé Attributes and Methods#

  • üìê shape ‚Üí dataset dimensions (üè≠ plants √ó üìÖ days).

  • üîÑ reshape ‚Üí reorganizes values into a new structure.

  • üìä max ‚Üí finds the peak production value.

  • üéØ argmax ‚Üí gives the index of that max value (in a flat array).

  • üó∫Ô∏è np.unravel_index ‚Üí converts flat index ‚Üí (row, column), so you know which plant üè≠ and which day üìÖ produced the maximum.

üëâ Useful for spotting peak energy output days üåûüí®üíß and reshaping datasets for analysis.

# Energy production data (12 values) in MWh
energy_data = np.arange(120, 132)   # 120,121,...,131
energy_matrix = energy_data.reshape(3, 4)  # 3 plants √ó 4 days

print(energy_matrix)
print("Shape:", energy_matrix.shape)        # 3 rows (plants) √ó 4 cols (days)
print("Reshape:", energy_matrix.reshape(2, 6))  # reshape into 2√ó6
print("Max:", energy_matrix.max())          # highest production value
print("Index of Max:", energy_matrix.argmax())  # index (flattened) of max

# Convert flat index to row & column
row, col = np.unravel_index(energy_matrix.argmax(), energy_matrix.shape)
print(f"Max production at Plant {row}, Day {col}")

‚ö° Type#

  • dtype ‚Üí tells you the data type of elements inside a NumPy array.

  • Common types:

    • int (integers, e.g. counts, whole kWh values)

    • float (decimals, e.g. efficiency, percentages)

    • bool (True/False, e.g. is_renewable)

  • NumPy arrays are homogeneous ‚Üí all elements must have the same type (unlike Python lists).

üëâ dtype is critical in analytics because it affects memory usage üíæ and precision üéØ.

# Daily solar energy production (kWh)
solar = np.array([120, 135, 150, 160])

print(solar)
print("Data type:", solar.dtype)

[120 135 150 160]
Data type: int64

# Efficiency values (as percentages)
efficiency = np.array([0.85, 0.90, 0.88, 0.92])

print(efficiency)
print("Data type:", efficiency.dtype)

[0.85 0.9  0.88 0.92]
Data type: float64

üéØ Indexing and Selection#

  • arr[0] ‚Üí first element (Day 1 üåû).

  • arr[1:4] ‚Üí slice Days 2‚Äì4 üìÖ.

  • arr[-1] ‚Üí last element (most recent day ‚è≥).

  • arr[-2:] ‚Üí last 2 days of production üîã.

üëâ Negative indexing is super handy when you want to quickly access the latest data points in time series.

# Energy production (in MWh) from a solar plant over 5 days
production = np.array([100, 120, 140, 160, 180])

print(production[0])     # First day's production
print(production[1:4])   # Days 2‚Äì4 production
print(production[-1])     # Last day's production
print(production[-2:])    # Last 2 days' production

üì° Broadcasting#

NumPy allows broadcasting: applying operations to arrays of different shapes or applying the same operation to all elements.

No need for loops ‚Üí fast & clean üöÄ.

# Base daily production values in MWh
production = np.array([100, 120, 130])

# Apply a constant 10 MWh bonus to each day
adjusted_production = production + 10

print(adjusted_production)

[110 130 140]

consumption = np.array([100, 120, 130])
increase = consumption * 1.05
print(increase)

[105.  126.  136.5]

üìù The Importance of copy()#

‚ö†Ô∏è In NumPy, simple assignments don‚Äôt create a new array, they create a view. To avoid this, use .copy().

solar = np.array([12, 15, 18, 20])
backup = solar
backup[0] = 99
print('Solar after backup change:', solar)

solar = np.array([12, 15, 18, 20])
safe_copy = solar.copy()
safe_copy[0] = 99
print('Solar:', solar)
print('Safe copy:', safe_copy)

Solar after backup change: [99 15 18 20]
Solar: [12 15 18 20]
Safe copy: [99 15 18 20]

üè¢ Matrices#

  • arr[row, col] ‚Üí pick a single value (plant √ó day).

  • arr[2] ‚Üí all columns for row 2 (Hydro plant üíß).

  • arr[:, 1] ‚Üí all rows for column 1 (all plants‚Äô production on Day 2 üìÖ).

# Energy production (MWh) for 3 plants over 4 days
# Rows = plants, Columns = days
production = np.array([
    [100, 120, 130, 140],  # Solar üåû
    [200, 210, 190, 220],  # Wind üí®
    [300, 310, 305, 295]   # Hydro üíß
])

print(production[0, 0])   # First plant, first day (Solar, Day 1)
print(production[1, 2])   # Second plant, third day (Wind, Day 3)
print(production[2])      # All days for Hydro plant
print(production[:, 1])   # All plants on Day 2
print('Shape:', production.shape)

‚ö° Matrix Multiplication#

  • Production matrix ‚Üí MWh produced (plants √ó days).

  • Cost vector ‚Üí price per MWh for each plant.

  • Matrix multiplication (np.dot) ‚Üí gives daily revenues across all plants.

üëâ This shows how linear algebra in NumPy can be directly applied to energy economics üí∂‚ö°.

production = np.array([
    [120, 130, 125],   # Solar üåû
    [200, 210, 205],   # Wind üí®
    [300, 310, 295]    # Hydro üíß
])

# Cost per MWh for each plant (‚Ç¨/MWh)
cost_per_mwh = np.array([50, 40, 30])  # Solar=50‚Ç¨, Wind=40‚Ç¨, Hydro=30‚Ç¨

# Total revenue per day = cost √ó production
# Matrix multiplication: (1x3 costs) √ó (3x3 production) = (1x3 revenues per day)
revenue = np.dot(cost_per_mwh, production)

print("Production matrix (MWh):\n", production)
print("Cost per MWh (‚Ç¨):", cost_per_mwh)
print("Revenue per day (‚Ç¨):", revenue)

Production matrix (MWh):
 [[120 130 125]
 [200 210 205]
 [300 310 295]]
Cost per MWh (‚Ç¨): [50 40 30]
Revenue per day (‚Ç¨): [23000 24200 23300]

# Using @ operator (preferred for readability)
revenue_at = cost_per_mwh @ production

print("Revenue with np.dot (‚Ç¨):", revenue)
print("Revenue with @ (‚Ç¨):", revenue_at)

Revenue with np.dot (‚Ç¨): [23000 24200 23300]
Revenue with @ (‚Ç¨): [23000 24200 23300]

üìä Statistics#

  • üìä Mean (np.mean) ‚Üí the average production across days.

  • üìâ Standard Deviation (np.std) ‚Üí how much production varies from the average (higher = more fluctuations).

  • üìê Variance (np.var) ‚Üí the squared measure of spread (basis of std dev).

# Daily solar energy production (MWh) over 4 days
production = np.array([100, 120, 130, 150])

print("Mean:", np.mean(production))      # üìä Average daily production
print("Std Dev:", np.std(production))    # üìâ Variation in production
print("Var:", np.var(production))        # üìê Spread of production

Mean: 125.0
Std Dev: 18.027756377319946
Var: 325.0

üß≠ Axes#

  • axis=0 ‚Üí down the rows (üìÖ column-wise)

    • Adds values for each column.

    • Here: total energy produced by all plants per day.

    • [320, 340, 330] ‚Üí Day 1 total = 320 MWh, Day 2 = 340, Day 3 = 330.

  • axis=1 ‚Üí across the columns (üè≠ row-wise)

    • Adds values for each row.

    • Here: total energy produced by each plant over all days.

    • [360, 630] ‚Üí Solar total = 360 MWh, Wind total = 630.

üëâ Think of axis like a direction:

  • axis=0 ‚Üí work down the rows.

  • axis=1 ‚Üí work across the columns.

energy_matrix = np.array([
    [120, 130, 110],
    [200, 210, 220]
])

print('Daily totals:', np.sum(energy_matrix, axis=0))
print('Plant totals:', np.sum(energy_matrix, axis=1))

Daily totals: [320 340 330]
Plant totals: [360 630]

üßÆ Universal Functions (ufuncs)#

NumPy provides many built-in mathematical functions that apply element-wise to arrays (no loops needed üöÄ). Common ones include:

  • np.exp ‚Üí exponential

  • np.log ‚Üí natural logarithm

  • np.sin, np.cos ‚Üí trigonometric functions

‚úÖ This creates a sine wave curve where:

  • Night hours ‚Üí 0 (no solar).

  • Noon ‚Üí peak production.

  • Evening ‚Üí returns to 0.

import matplotlib.pyplot as plt

# Hours in a day (0‚Äì23)
hours = np.arange(0, 24)

# Model solar production using a sine wave
# Shifted so it's zero at night and peaks at noon
solar_output = np.maximum(0, np.sin((hours - 6) * np.pi / 12)) * 100

print("Solar output (MWh):", solar_output)

# Plot the solar cycle
plt.plot(hours, solar_output, marker='o')
plt.title("Solar Energy Production Over a Day")
plt.xlabel("Hour of Day")
plt.ylabel("Production (MWh)")
plt.grid(True)
plt.show();
